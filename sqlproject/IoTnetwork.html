IoT Network Traffic Analysis for Anomaly Detection

Objective:

My project focused on analysing network traffic data from IoT devices with the goal of detecting anomalies and potential threats. The data was processed using advanced SQL techniques to clean, balance, and enhance the dataset, which laid the groundwork for building a robust Intrusion Detection System (IDS).

Data Preprocessing and Analysis:

Data Cleaning:
- Filling missing packet and byte information with zeroes.
- Checking for and handling any null values in key fields.
- Deleting duplicate entries to maintain dataset accuracy.

After cleaning, I moved to Data Balancing to counteract the class imbalance that often plagues anomaly detection tasks. This was achieved by sampling equal instances of 'Malicious' and 'Benign' labels, creating a balanced working dataset for reliable model training.

Feature Engineering was a creative phase where I introduced new attributes like a boolean column to flag common IoT ports. Additionally, I calculated the average bytes per packet to understand the traffic patterns more effectively. These engineered features were critical in identifying unusual traffic that could signify security risks.

For Feature Scaling, I performed a min-max normalization on orig_ip_bytes, a key step for models that are sensitive to the scale of the data.

Categorical Variables Encoding was applied to the protocol field to transform textual data into a numeric format, preparing the dataset for algorithmic processing.

Next, I divided the dataset into training and testing subsets during the Data Partitioning Stage . This is an essential step for model validation and to avoid overfitting.

Analytical Insights:

- Data Aggregation: Leveraged powerful aggregation functions like COUNT(), SUM(), and AVG() to summarise and analyze data, providing insights into the distribution of traffic and identifying potential imbalances.
- Conditional Logic: Utilised CASE statements to encode categorical variables, transforming textual protocol information into numerical codes suitable for analysis.
- Data Definition & Manipulation: Employed ALTER TABLE to enhance the dataset structure by adding new columns, and UPDATE commands to populate these columns with derived data for feature enrichment.
- Window Functions: Deployed advanced window functions such as LEAD(), LAG(), ROW_NUMBER(), and RANK() to detect anomalies and understand sequential traffic patterns, which are vital in predicting network intrusions.
- Random Sampling: Implemented the RANDOM() function in conjunction with CREATE TEMP TABLE statements to partition the dataset into training and testing subsets, which is crucial for the validation and testing of machine learning models.
- Data Normalisation: Conducted a basic form of feature scaling by applying min-max normalization directly within SQL, making the numerical features ready for algorithms that are sensitive to feature magnitude.

True Negatives (TN):The model correctly identified 965 non-spam messages. This is a high number and suggests that the model is very effective at identifying the majority of genuine messages.

Analysis Overview

Traffic Analysis: We assessed traffic volume and behavior to identify anomalies and discern trends.
Protocol and Port Analysis: We performed a granular inspection of protocol usage and port activities, unearthing potential areas of risk.
Port-Specific Analysis: We evaluated traffic patterns per port, highlighting those exhibiting irregular behaviors.
